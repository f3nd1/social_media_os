HANDOVER PACK

UCC Marketing OS

A guide to understanding and testing the app

This pack explains, in plain language, what the marketing operations app for United Ceres College does, how to get it running, and exactly what to try. No coding knowledge is needed.

An honest note before you start. This is a working prototype, not a finished product. Some parts need a connection key or a live test with real accounts before they are fully proven. Section 5 lists what is not finished, so nothing here overstates how ready it is.
Prepared 5 July 2026. Runs on your own computer for now (it is not yet on a public web address).
Section 1

What this system is

UCC Marketing OS is a marketing operations app for United Ceres College. It is objective-first, which means every activity is tied to a measurable goal. It runs on one simple principle: AI recommends, humans decide. The AI only ever writes drafts. A person reviews and approves everything, and nothing is ever published or approved automatically.

What it does

  • Plan marketing campaigns, each tied to a clear objective
  • Generate content calendars and copy with AI, always as drafts you approve
  • Check wording against education marketing compliance rules
  • Track KPIs and real results against targets
  • Produce weekly reports and a permanent approvals record for management

Section 3

How to get it running

You do not need to write any code. These are the only steps:

  • Make sure the computer has Node.js version 20 or newer installed (a free tool for running the app).
  • Get the code: the repository is f3nd1/social_media_os. Use the main branch.
  • Open a terminal (a text command window) in the project folder.
  • Type npm install and press enter. This sets things up once. Wait for it to finish.
  • Type npm run dev and press enter. This starts the app.
  • Open a web browser and go to http://localhost:3000
  • To stop the app, return to the terminal and press Ctrl and C together.

Section 4

What to test

A. One end-to-end path (tick as you go)

  • Open Settings and click Run setup guide
  • Choose Set up my own
  • Connect Supabase, OpenAI and Metricool, or skip each (all optional)
  • Read the short compliance reminder and continue
  • Enter a brand name and tone of voice
  • Add at least one course
  • Add at least one audience
  • Record a quick audit for one platform
  • Generate or type a Strategy Brief, then approve it
  • Finish the guide and click Generate my first content calendar
  • Open the calendar, open an item, and approve it
  • Open the Dashboard and check the setup status and numbers have updated

Section 5

Known limitations and open items

Being straight about what is not finished:

  • Metricool gives brand-level totals, not per-network numbers. For per-platform detail, use the CSV import instead.
  • Platform connections other than Metricool store your keys but do not sync data yet. Use CSV or type numbers in for those.
  • Social listening needs a hosted background worker when the app is deployed. It may not run on a simple serverless host without one. The PDF text extraction and one compliance extraction route use similar small helpers.
  • The app is not yet on a public web address. It runs on your own computer for now.
  • There is no login and no user roles yet. Anyone with the link, and the Supabase key if that is connected, can read and write the data. Fine for a prototype, but not for real student data.
  • The live AI, Supabase and Metricool paths were checked by simulation during development. They still need a final test with real keys on a real machine.
  • The approver name is free text for now, so the audit trail records a typed name rather than a verified login.
Section 6

How to give feedback

If anything looks wrong, confusing, or missing, please note these four things. They make it quick to fix:

  • Which screen you were on
  • What you expected to happen
  • What actually happened
  • A screenshot if you can
One extra line that helps a lot: was AI connected (an OpenAI key added) at the time? Some buttons behave differently with and without a key, so knowing this saves a lot of guessing.

Thank you for testing. Your notes go straight into making it clearer and more reliable.
